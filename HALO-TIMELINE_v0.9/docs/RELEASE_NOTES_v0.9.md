# RELEASE NOTES — HALO-TIMELINE v0.9

## Overview
HALO-TIMELINE v0.9 は、Windows から取得した主要アーティファクト  
**(EVTX / LNK / JumpList / MFT / USNJ)** の CSV を **単一のタイムラインに統合**するための、
最小構成の PowerShell スクリプトです。

本バージョンは、より高度で複雑な構成を持つ v1.1 ではなく、  
**「まず動く。再現できる。」ことに焦点を当てた状態** をコアとして公開しています。

---

## Why This Version (v0.9)
HALO プロジェクトでは、ツールの高度化よりも **再現性** を優先します。

v1.1 では、抽出プロセス、依存ツール、出力整形などが拡張されていましたが、  
そのぶん「環境により動作が揺れる」「初学者が入りづらい」という懸念がありました。

そこで本リリースでは、  
**・依存の少なさ**  
**・構造の読みやすさ**  
**・一度動かせば理解が始まる設計**  
を守るため、あえて **v0.9 をコアとして採用**しています。

---

## Value of Degeneration (縮退の価値)

| 目的 / 値 | 説明 |
|---|---|
| **再現性** | 少ない構成は「環境差分による失敗」を防ぐ |
| **理解の歩幅** | 出力を見ながら「何が起きているか」を追いやすい |
| **教育可能性** | 学習会・自主学習・研修教材として流用できる |
| **運用の強さ** | 余計な抽象化を排除した構造は、壊れにくい |

v0.9 は「必要最小限で動作を定義する」ことで、  
**どの環境でも、誰が触っても、同じ結果に辿り着ける** ことを保証します。

これは「機能が少ない」という意味ではなく、  
**土台が強い** という設計思想です。

---

## Intended Audience
- DFIR 初学者 / 自主学習者
- セキュリティ教育 / 社内研修
- 実務で迅速にタイムラインを得る必要があるアナリスト
- 既存ツールの動作理解を深めたい人

---

## Project Philosophy (短く、ただ一行)
> **First, get it to run once. Understanding starts from there.**

理解は「手を動かしたあと」に宿ります。  
HALO は、その最初の一歩を支えるためのプロジェクトです。

---

## Change Summary
- 統合ロジックを **単一スクリプト** に集約
- 出力形式を **固定列・UTC ISO8601** に統一
- examples/ に **最小の合成タイムライン例** を追加
- 分析前提の複雑な処理は **あえて導入しない**

---

## Next Steps
v1.x 系では、以下を段階的に再導入していきます：

- 解析対象の自動抽出
- タイムライン拡張 (SRUM, ShimCache, Amcache, etc.)
- GUI / Jupyter / Obsidian 連携機能

ただし、**「再現性に影響する複雑化」より優先はしません。**
